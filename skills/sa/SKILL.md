# Skill & Agent Advisor

사용자 입력: **$ARGUMENTS**

인자가 없으면 "어떤 상황인지 간단히 말씀해주세요" 하고 먼저 물어보세요.

## 실행 절차

### Step 1: 활성화된 스킬 + 에이전트 목록 수집

**1a. 스킬 수집:** 현재 대화의 system-reminder 중 **"The following skills are available for use with the Skill tool:"** 블록을 찾아서, 거기에 나열된 모든 스킬의 **이름**과 **description**을 파악하세요.

**1b. 에이전트 수집:** `~/.claude/agents/` 디렉토리의 `.md` 파일들을 스캔하세요. 각 파일의 YAML frontmatter에서 `name`, `description`, `model`을 읽어 에이전트 목록을 구성합니다.

```bash
for f in ~/.claude/agents/*.md; do head -10 "$f" | grep -E '^(name|description|model):'; echo "---"; done
```

스킬과 에이전트를 합친 목록이 추천 대상의 전체 범위입니다.

### Step 2: 상황 매칭

`$ARGUMENTS`의 의도를 파악한 뒤, Step 1에서 수집한 스킬과 에이전트의 description을 하나씩 대조하여 가장 적합한 것을 선택하세요.

매칭 우선순위:
1. description이 사용자 상황과 직접적으로 일치하는 스킬/에이전트
2. 이름(name)이 상황 키워드와 관련 있는 스킬/에이전트
3. 여러 스킬/에이전트를 조합하면 시너지가 나는 경우 함께 제시
4. **스킬과 에이전트를 함께 조합** 가능 (예: `/investigate` 스킬 + `postgres-pro` 에이전트)

### Step 3: 추천 출력

상황 복잡도에 따라 **1~4개**를 유동적으로 추천하세요.
- 단순한 작업 → 1~2개
- 복합적인 작업 → 3~4개
- 각 추천에 순서 번호를 매기고, 순차 실행이 효과적이면 **워크플로우 순서**를 명시

아래 형식으로 답변하세요.

---

**상황**: [한 줄 요약]

**① 메인**: `[명령어 또는 에이전트명]` `[스킬]`/`[에이전트]`
[이유 1줄]

**② 보조**: `[명령어 또는 에이전트명]` `[스킬]`/`[에이전트]`
[이유 1줄]

**③ 추가**: `[명령어 또는 에이전트명]` `[스킬]`/`[에이전트]` *(해당 시에만)*
[이유 1줄]

**④ 참고**: `[명령어 또는 에이전트명]` `[스킬]`/`[에이전트]` *(해당 시에만)*
[이유 1줄]

**워크플로우**: ① → ② → ③ → ④ 순서와 각 단계의 연결 이유

---

**진행하시겠습니까?**
- `y` 또는 "진행" → 워크플로우 순서대로 전체 실행
- `1`, `2`, `3`, `4` → 해당 번호만 단독 실행
- `1,3` 등 콤마 조합 → 선택한 것만 순서대로 실행

### Step 4: 진행 처리

사용자가 진행 의사를 밝히면 아래 규칙에 따라 실행하세요.

**실행 방법 — 스킬 vs 에이전트:**
- **스킬**: Skill 도구로 호출 (예: `Skill("investigate")`)
- **에이전트**: Agent 도구로 호출 (예: `Agent(subagent_type: "postgres-pro", prompt: "사용자의 요청 내용")`)

**전체 실행** (`y`, "진행", "시작", "go", "ok" 등):
1. "워크플로우로 진행합니다: ① → ② → ③ → ④" 안내 (추천된 것만)
2. ① 메인을 적절한 도구(Skill 또는 Agent)로 호출
3. 완료 후 "다음으로 ② 실행합니다" 안내
4. 나머지 번호를 순서대로 반복

**선택 실행** (`1`, `2`, `3`, `4` 또는 콤마 조합 `1,3`):
- 지정된 번호만 순서대로 실행
- 예: `1,3` → ①번 실행 후 ③번 실행

**단독 실행** (번호 하나만):
- 해당 번호만 적절한 도구로 호출

> ⚠️ 추천이 1개뿐인 경우: "진행"하면 해당 항목만 실행.
